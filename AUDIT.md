## Summary
Nelover is a single-file static landing page (index.html, ~27 KB) using Three.js r134 + GSAP 3.12.2 from cdnjs with no local fallback or Subresource Integrity. The code runs and the site renders given internet access, so there is no broken-build defect, but the repository is in a precarious state: a 63-file destructive migration (the old React/Vite app) sits deleted-but-uncommitted with no .gitignore, the README points at a stale absolute path, ~2.6 MB of unoptimized PNGs ship to every visitor, and accessibility basics (reduced-motion, hidden cursor, SEO meta) are missing. No secrets are present. Issues are maintainability, performance, and hygiene rather than security. Overall grade: C

## High
### [H1] Destructive React/Vite removal staged but uncommitted, with no .gitignore
- Where: git working tree (root); `git status --short` = 70 entries, 63 of them `D` (deletions)
- Evidence: package.json, package-lock.json, public/, src assets, eslint configs, LICENSE, and .gitignore all show as `D`; HEAD/`git log` still describe the old React app ("Fix navigation links to use React Router Link"); the new site (index.html mods, assets/, CLAUDE.md, .claude/, patch scripts) is untracked/modified and never committed.
- Impact: One `git checkout .` or `git restore .` wipes the entire current site and resurrects the dead React app; nothing is committed as a recovery point. No .gitignore means future `node_modules`, `__pycache__`, or editor cruft will be tracked by default.
- Fix: Commit the migration as an intentional snapshot: `git add -A && git commit -m "Migrate to single-file static terrarium site"`. Add a `.gitignore` (node_modules/, __pycache__/, .DS_Store, *.pyc) before staging.

## Medium
### [M1] CDN dependencies loaded without Subresource Integrity or offline fallback
- Where: index.html:625-627 (three.js r134, gsap 3.12.2, ScrollTrigger) and :9-12 (Google Fonts)
- Evidence: `<script src="https://cdnjs.cloudflare.com/.../three.min.js"></script>` with no `integrity=`/`crossorigin=` attributes; no local copy.
- Impact: With no internet (or if cdnjs is down/compromised) the page is blank — Three.js and GSAP are referenced unconditionally in the inline script, so a load failure throws ReferenceError and halts all JS. No SRI means a tampered CDN file would execute silently.
- Fix: Add `integrity` + `crossorigin="anonymous"` hashes to each `<script>`, or vendor the libraries into a local `vendor/` directory and reference them relatively.

### [M2] ~2.6 MB of unoptimized PNGs shipped to every visitor
- Where: assets/about_garden.png (710 KB), hero_xl.png (701 KB), hero_woodland.png (684 KB), hero_moss.png (590 KB)
- Evidence: `ls -la assets/` totals ~2.6 MB across 4 PNGs; all are referenced by `<img>` in index.html (:670, :706, :724, :741) with no `loading="lazy"`, `srcset`, or width/height.
- Impact: Slow first paint and high bandwidth on mobile; these images are decorative product shots that do not need lossless PNG.
- Fix: Convert to WebP/AVIF (typically 80-90% smaller), add `loading="lazy"` + intrinsic `width`/`height` to prevent layout shift, and provide `srcset` for responsive sizes.

### [M3] No prefers-reduced-motion guard around continuous animation
- Where: index.html:939-954 (infinite `requestAnimationFrame` render loop), GSAP ScrollTriggers (:957-999), CSS `scroll-behavior: smooth` (:38) and transitions throughout
- Evidence: No `@media (prefers-reduced-motion: reduce)` rule anywhere; the 3D scene animates and parallaxes on every frame regardless of user setting.
- Impact: Accessibility regression for motion-sensitive users (vestibular disorders); also a constant battery/CPU drain since the rAF loop never pauses even when the canvas is offscreen.
- Fix: Wrap the rAF loop and GSAP triggers behind `window.matchMedia('(prefers-reduced-motion: reduce)')`; disable smooth scroll and scrub animations when reduce is set. Optionally pause rendering when the tab is hidden (`document.visibilityState`).

### [M4] Missing SEO/social meta and real favicon
- Where: index.html:5-8 (head)
- Evidence: `<link rel="icon" href="data:,">` (intentionally blank favicon → 404-style empty icon), no `<meta name="description">`, no Open Graph (`og:title`/`og:image`/`og:description`) or Twitter card tags.
- Impact: Poor search-result snippets and blank/ugly previews when the landing page is shared on social media — a real drawback for a marketing site.
- Fix: Add a meta description, Open Graph + Twitter card tags (use one of the hero PNGs, optimized, as `og:image`), and ship a real favicon.

### [M5] One-off mutation scripts left in repo with stale hardcoded paths
- Where: patch_html.py:4 and patch2.py:3
- Evidence: Both begin `filepath = '/media/eyad/app/projects/Nelover/index.html'` and perform `str.replace` rewrites of index.html that are already applied; the two files are near-duplicates (both define the same scripts block, resize handler, etc.).
- Impact: Dead weight that is confusing and dangerous — re-running them would either fail (path no longer exists at /media/eyad/...) or, if pointed at the current file, double-apply/corrupt index.html. They imply an edit history that no longer matches reality.
- Fix: Delete both scripts (their changes are baked into index.html). If kept for reference, move to a `scripts/archive/` folder with a README note that they must not be run.

### [L2] `cursor: none` with a JS-only custom cursor hurts accessibility
- Where: index.html:47 (`body { cursor: none }`) and :790-812 (JS cursor)
- Evidence: The native cursor is hidden globally; the replacement is two divs positioned via `mousemove`. If JS fails or on touch devices, there is no visible pointer.
- Impact: Users relying on the system cursor (or with JS disabled / on hover-incapable devices) lose pointer feedback; the gold dot also has lower contrast/visibility than a native cursor.
- Fix: Restore a native cursor under `@media (hover: none)` and `prefers-reduced-motion`, or at minimum keep `cursor: auto` as a fallback if the custom-cursor JS does not initialize.

### [L3] Decorative `<img>` tags download even when hidden on error
- Where: index.html:670, 706, 724, 741 (`onerror="this.style.display='none'"`)
- Evidence: `onerror` only hides the element *after* a failed/successful fetch — the browser still requests the PNG regardless.
- Impact: Bandwidth is spent fetching images that may be hidden; `onerror=display:none` is a band-aid, not a load-avoidance strategy.
- Fix: Pair with `loading="lazy"` (see M2) so offscreen images are deferred, and rely on optimized sources rather than hide-on-error.

### [L4] No WebGL context-loss handling
- Where: index.html:825-1007 (init3D IIFE); resize handler at :1002 only
- Evidence: The renderer is created once; there is no `webglcontextlost`/`webglcontextrestored` listener.
- Impact: On a GPU reset or context loss (common on mobile/integrated GPUs), the canvas goes blank and never recovers without a page reload.
- Fix: Add a `canvas.addEventListener('webglcontextlost', ...)` handler that prevents default and re-initializes the scene on restore.

## Optimizations
- Inline CSS/JS (~27 KB total) is fine for a single page, but if the site grows, split inline `<style>`/`<script>` into cacheable external files so the HTML document itself stays small and the libraries/styles can be browser-cached across navigations.
- Add `rel="preconnect"`/`dns-prefetch` for `cdnjs.cloudflare.com` (currently only fonts.googleapis.com is preconnected) to shave CDN handshake latency.
- Particle count is tiny (50) and geometry is cheap, but capping `setPixelRatio` at 2 is already good; consider pausing the rAF loop when the canvas/scene is fully scrolled out of view to save power.
- Reduce Google Fonts payload by trimming unused weights (e.g. Cormorant italic variants) if they are not all used in the markup.
